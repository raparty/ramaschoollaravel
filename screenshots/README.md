## Design System

### Color Palette
- **Primary (Azure Blue)**: #0078D4 - Headers, primary actions, links
- **Secondary (Slate)**: #605E5C - Text, borders, secondary elements
- **Background**: #FAF9F8 - Page backgrounds, cards
- **Success**: #107C10 - Success states, confirmations
- **Warning**: #F7630C - Warnings, important actions
- **Danger**: #D13438 - Errors, deletions

### Layout Architecture
The application uses Bootstrap 5 with custom enhancements:
- **Responsive Grid**: Mobile-first design with breakpoints
- **Flexbox Navigation**: Fixed header with collapsible sidebar
- **Card-based Layouts**: Consistent spacing and shadows
- **Print Optimization**: Dedicated print stylesheets for receipts and reports

## Adding New Screenshots

To add screenshots for new features:

1. Set up the application with sample data
2. Navigate to the desired module
3. Capture at 1920x1080 resolution (or responsive view)
4. Save as PNG with descriptive filename
5. Add entry to this README with description
6. Update main README.md to reference the screenshot

## Screenshot Guidelines

- **Resolution**: Minimum 1920x1080 for desktop views
- **Format**: PNG for quality
- **Content**: Include representative data (not empty states)
- **Privacy**: Use fake/sample data only
- **Naming**: Use descriptive, lowercase names with underscores
- **Size**: Optimize images (max 500KB per screenshot)
